# สรุปโครงการ: การปรับปรุงแอปพลิเคชันร้านอาหารแบบ Fullstack

เอกสารนี้สรุปการเปลี่ยนแปลงและการแก้ไขข้อผิดพลาดที่นำไปใช้ในแอปพลิเคชันร้านอาหารแบบ fullstack

## 1. การปรับปรุงหน้าแรก (Home Page)

- **วัตถุประสงค์:** แสดงข้อมูลร้านอาหารจาก `db.json` บนหน้าแรก
- **การนำไปใช้:** แก้ไข `client/src/pages/Home.jsx` เพื่อดึงข้อมูลร้านอาหารโดยตรงจาก `/db.json` แทนที่จะพึ่งพา API ของฝั่งเซิร์ฟเวอร์

## 2. การแสดงรูปภาพสำหรับร้านอาหาร

- **วัตถุประสงค์:** ตรวจสอบให้แน่ใจว่ารูปภาพร้านอาหารแสดงบนหน้าแรก
- **การนำไปใช้:**
    - แก้ไข `client/src/components/Restaurants.jsx` เพื่อส่งคุณสมบัติ `img` (จาก `db.json`) เป็น `imageUrl` prop ไปยังคอมโพเนนต์ `Card` อย่างถูกต้อง
    - ยืนยันว่า `client/src/components/Card.jsx` ได้รับการตั้งค่าให้แสดงรูปภาพโดยใช้ `imageUrl` prop อยู่แล้ว

## 3. หน้าที่ทำงานได้ (Add, Update, Login, Register)

- **วัตถุประสงค์:** ทำให้หน้า `Add.jsx`, `Update.jsx`, `Login.jsx` และ `Register.jsx` ทำงานได้อย่างสมบูรณ์สำหรับการดำเนินการ CRUD และการยืนยันตัวตนผู้ใช้

### 3.1. Add.jsx
- **วัตถุประสงค์:** เปิดใช้งานการเพิ่มรายการร้านอาหารใหม่
- **การนำไปใช้:**
    - กำหนดค่า `client/src/pages/Add.jsx` เพื่อส่งคำขอ `POST` ไปยัง endpoint `/api/v1/restaurants` ของเซิร์ฟเวอร์
    - ตรวจสอบความสอดคล้องของฟิลด์ข้อมูล (`title`, `type`, `img`)
    - ใช้การจัดการฟอร์ม (`handleChange`, `handleSubmit`) และการเปลี่ยนเส้นทางไปยังหน้าแรกเมื่อเพิ่มสำเร็จ

### 3.2. Update.jsx
- **วัตถุประสงค์:** อนุญาตให้อัปเดตรายการร้านอาหารที่มีอยู่
- **การนำไปใช้:**
    - แก้ไข `client/src/pages/Update.jsx` เพื่อดึงข้อมูลร้านอาหารเฉพาะจาก `/db.json` ตาม `id` ของ URL เพื่อกรอกข้อมูลในฟอร์มล่วงหน้า
    - ตรวจสอบความสอดคล้องของฟิลด์ข้อมูล (`title`, `type`, `img`)
    - กำหนดค่าให้ส่งคำขอ `PUT` ไปยัง endpoint `/api/v1/restaurants/:id` ของเซิร์ฟเวอร์สำหรับการอัปเดต

### 3.3. Register.jsx
- **วัตถุประสงค์:** ใช้ฟังก์ชันการลงทะเบียนผู้ใช้
- **การนำไปใช้:**
    - กำหนดค่า `client/src/pages/Register.jsx` เพื่อส่งคำขอ `POST` ไปยัง endpoint `/api/v1/register` ของเซิร์ฟเวอร์
    - ใช้การจัดการฟอร์มสำหรับ `username`, `fullName`, `email` และ `password`
    - เพิ่มการเปลี่ยนเส้นทางไปยังหน้าเข้าสู่ระบบเมื่อลงทะเบียนสำเร็จ

### 3.4. Login.jsx
- **วัตถุประสงค์:** ใช้ฟังก์ชันการเข้าสู่ระบบของผู้ใช้
- **การนำไปใช้:**
    - กำหนดค่า `client/src/pages/Login.jsx` เพื่อส่งคำขอ `POST` ไปยัง endpoint `/api/v1/signin` ของเซิร์ฟเวอร์
    - ใช้การจัดการฟอร์มสำหรับ `username` และ `password`
    - จัดเก็บโทเค็นการยืนยันตัวตนใน `localStorage` เมื่อเข้าสู่ระบบสำเร็จ
    - เพิ่มการเปลี่ยนเส้นทางไปยังหน้าแรกเมื่อเข้าสู่ระบบสำเร็จ

## 4. การแก้ไขข้อผิดพลาด

ในระหว่างกระบวนการพัฒนา ได้มีการระบุและแก้ไขข้อผิดพลาดหลายประการ:

- **`Add.jsx`: ไม่มี `catch` block:** แก้ไข `SyntaxError` โดยการเพิ่ม `catch` block ในฟังก์ชัน `handleSubmit` เพื่อจัดการข้อผิดพลาดระหว่างการเพิ่มร้านอาหารอย่างเหมาะสม
- **`Add.jsx`: ตำแหน่ง `return` statement ผิด:** แก้ไข `SyntaxError` โดยการย้าย `return` statement หลักของคอมโพเนนต์ออกนอกฟังก์ชัน `handleSubmit`
- **`Login.jsx`: ไม่มี `default export`:** แก้ไข `Uncaught SyntaxError` โดยการเพิ่ม `export default Login;` ไปยังไฟล์ `client/src/pages/Login.jsx` เพื่อให้แน่ใจว่าคอมโพเนนต์ถูกส่งออกอย่างถูกต้องสำหรับการกำหนดเส้นทาง

## 5. การพยายามจัดสไตล์และการย้อนกลับ

- **วัตถุประสงค์:** พยายามปรับปรุงรูปลักษณ์ของ `Add.jsx`, `Update.jsx`, `Login.jsx`, และ `Register.jsx` โดยใช้ DaisyUI และ Tailwind CSS
- **ผลลัพธ์:** เนื่องจากข้อจำกัดของเครื่องมือ `replace` อัตโนมัติสำหรับการเปลี่ยนแปลงสไตล์ที่ซับซ้อน สไตล์ที่ใช้จึงถูกย้อนกลับไปเป็นรูปลักษณ์เดิมของหน้าเว็บตามคำขอของผู้ใช้ การใช้คลาสสไตล์ด้วยตนเองจะต้องดำเนินการสำหรับการปรับปรุงความสวยงามเพิ่มเติม
